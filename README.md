# hw1_maze

Проект на Unreal Engine с актором `Pathfinder`, который строит дискретную сетку поверх уровня, помечает занятые клетки как стены и ищет путь от `StartPoint` до `FinishPoint` алгоритмом `Dijkstra` или `A*`.

## Как работает код

При старте игры `APathfinder::BeginPlay()` последовательно выполняет три шага:

1. Собирает сетку через `BuildGrid()`.
2. Находит стартовую и конечную клетки через `ResolveStartAndFinish()`.
3. Запускает поиск пути через `FindPathDijkstra()` или `FindPathAStar()`.

### Как получается сетка

- Размер сетки задают `GridWidth`, `GridHeight` и `CellSize`.
- Левый нижний угол сетки в мире задается через `GridOrigin`.
- Для каждой клетки `(X, Y)` создается `FGridNode`.
- Центр клетки переводится в мировые координаты функцией `GridToWorld()`.
- Индекс узла хранится в одномерном массиве `GridNodes` по формуле `Y * GridWidth + X`.

### Как определяются стены

- Для каждой клетки вызывается `CheckWallAtCell(X, Y)`.
- В центре клетки выполняется `OverlapMultiByChannel` с коробкой размера `WallCheckHalfExtent x WallCheckHalfExtent x WallCheckHalfHeight`.
- Если среди пересечений есть актор с тегом `Wall`, клетка помечается как непроходимая: `bWalkable = false`.
- Если тега `Wall` нет, клетка считается свободной.

### Как выбираются соседи

- Соседи клетки считаются в `GetNeighborIndices()`.
- Используются только 4 направления: вправо, влево, вверх и вниз.
- Диагоналей нет.
- В список соседей попадают только клетки внутри сетки и только проходимые клетки.

### Как работает поиск пути

- `ResolveStartAndFinish()` переводит позиции `StartPoint` и `FinishPoint` из мира в координаты сетки через `WorldToGrid()`.
- Если старт или финиш вне сетки либо попадают в стену, поиск не запускается.
- В `Dijkstra` каждая дуга имеет стоимость `1`, и алгоритм всегда расширяет узел с минимальной накопленной стоимостью `Cost`.
- В `A*` используется та же стоимость перехода `1`, но дополнительно считается эвристика `HeuristicCost` как Manhattan distance до финиша.
- Лучший узел в открытом списке выбирается в `GetBestOpenSetIndex()`.
- После достижения финиша путь восстанавливается назад по `ParentIndex` в `ReconstructPath()`.

### Что показывает debug

- `DrawGridDebug()` рисует клетки сетки: свободные серым, стены красным.
- `DrawStartFinishDebug()` рисует старт зеленым и финиш синим.
- `DrawSearchStateDebug()` показывает open set желтым и closed set оранжевым.
- `DrawPathDebug()` рисует найденный путь зелеными линиями и точками.
- Если включен `bAnimateSearchDebug`, визуализация показывается пошагово через `UpdateDebugAnimation()`.

## Основные параметры

- `GridOrigin` - начало сетки в мировых координатах.
- `GridWidth` / `GridHeight` - размеры сетки в клетках.
- `CellSize` - размер одной клетки.
- `WallCheckHalfExtent` - половина ширины проверки препятствия внутри клетки.
- `WallCheckHalfHeight` - половина высоты проверки препятствия.
- `DebugZOffset` - высота debug-отрисовки над уровнем.
- `StartPoint` - точка старта.
- `FinishPoint` - точка финиша.
- `SearchAlgorithm` - выбор между `Dijkstra` и `A*`.

## Debug-параметры

- `bDrawGridDebug` - рисует сетку.
- `bDrawOnlyBlockedCells` - оставляет только заблокированные клетки.
- `bDrawStartFinishDebug` - рисует старт и финиш.
- `bDrawPathDebug` - рисует итоговый путь.
- `bDrawSearchStateDebug` - рисует состояния поиска.
- `bAnimateSearchDebug` - включает пошаговую анимацию.
- `DebugIterationsPerSecond` - скорость анимации.

## Ограничения

- Старт и финиш должны попадать внутрь сетки.
- Старт и финиш не должны стоять в заблокированных клетках.
- Препятствия учитываются только если у актора есть тег `Wall`.
- Поиск идет только по ортогональным соседям, без диагоналей.
